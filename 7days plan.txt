7-Day Interview Prep Plan (Software / Tech)
Day 1: Arrays & Strings
Why: Most common topic in coding interviews.
Examples to practice:
Reverse a string
Two sum problem
Maximum subarray sum (Kadane’s algorithm)
Goal: Understand basic operations (traverse, search, sort, reverse).

Day 2: Linked Lists
Why: Shows knowledge of pointers/references.
Examples to practice:
Reverse a linked list
Detect cycle in linked list (Floyd’s algorithm)
Merge two sorted linked lists
Goal: Understand linked list traversal, insertion, deletion, and cycle detection.

Day 3: Stacks & Queues
Why: Fundamental data structures; used in many problems.
Examples to practice:
Valid parentheses problem
Implement stack using array/linked list
Queue using two stacks
Goal: Understand LIFO (stack) and FIFO (queue) behavior.

Day 4: Trees & Graphs
Why: Many system-level & algorithmic problems depend on trees/graphs.
Examples to practice:
Binary tree traversals (inorder, preorder, postorder)
Lowest common ancestor (LCA)
BFS/DFS in a graph
Goal: Understand hierarchical data structures and traversal algorithms.

Day 5: Sorting & Searching
Why: Core concept for problem solving.
Examples to practice:
Binary search in array
Merge sort / Quick sort
Search in rotated sorted array
Goal: Understand time complexity, different search & sort methods.


Day 6: Dynamic Programming (DP)
Why: Shows problem-solving depth; asked in medium/hard questions.
Examples to practice:
Fibonacci numbers (recursive + DP)
Coin change problem
Longest common subsequence
Goal: Understand memoization, tabulation, problem breakdown.

Day 7: System Design / Object-Oriented Programming
Why: Big tech & product-based interviews focus here.
Examples to practice:
Design a URL shortener (like bit.ly)
Design an online library system
Design a basic chat system
Goal: Practice designing scalable systems, classes, relationships, and trade-offs.
